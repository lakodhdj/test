# Полный фронтенд E-Commerce

## 📱 Обзор

Минималистичный веб-интерфейс для e-commerce приложения, разработанный на Vanilla JavaScript, HTML5 и CSS3. **Без любых зависимостей и фреймворков** - чистый, быстрый и легкий код.

## 🎨 Дизайн

### Минималистичный стиль
- **Нейтральная палитра**: Черный текст, белый фон, серые акценты
- **Типография**: Системные шрифты для быстрой загрузки
- **Отсутствие излишних элементов**: Только необходимый UI
- **Белое пространство**: Чистые компоновки с хорошим дыханием
- **Четкие границы**: Минимальная толщина линий

### Адаптивность
```
Desktop  (1200px+):  4 колонки товаров, полная навигация
Tablet   (768-1199px): 3 колонки товаров
Mobile   (<768px):   1 колонка товаров, компактное меню
```

## 🚀 Функции

### 1️⃣ Аутентификация

#### Регистрация
```javascript
POST /account/register
{
  "email": "user@example.com",
  "password": "password123"
}
```

**Возможности:**
- Валидация email
- Проверка длины пароля
- Обработка ошибок
- Успешное уведомление

#### Вход
```javascript
POST /account/login
{
  "email": "user@example.com",
  "password": "password123"
}
```

**Особенности:**
- httpOnly cookies для токенов
- Автоматическое обновление токенов
- Сохранение сессии
- Безопасность

#### Профиль
- Просмотр email и статуса верификации
- Смена пароля
- Выход из аккаунта

### 2️⃣ Каталог товаров

#### Просмотр товаров
- Сетка товаров (4/3/1 колонка в зависимости от экрана)
- Изображения товаров
- Название, цена, статус наличия
- Кнопки действий (корзина, избранное)

#### Фильтрация
```
Search: Поиск по названию
Category: Фильтр по категориям
Gender: Мужской / Женский / Унисекс
Size: XS / S / M / L / XL / XXL
Price: Диапазон от мин до макс
```

**Пример:**
```
Поиск: "рубашка"
Категория: "Одежда"
Пол: "Мужской"
Размер: "M"
Цена: 1000-5000₽
```

#### Детали товара
При клике на товар открывается модальное окно с:
- Полное описание
- Изображение (если есть)
- Полные характеристики
- Выбор количества
- Кнопка добавления в корзину
- Кнопка избранного

#### Пагинация
- 20 товаров на странице
- Навигация между страницами
- Сохранение фильтров при переходе

### 3️⃣ Корзина

#### Добавление
```javascript
POST /cart/{product_id}?quantity=1
```

**Процесс:**
1. Выбрать товар
2. Указать количество
3. Нажать "Добавить в корзину"
4. Товар добавляется в локальную корзину и на сервер

#### Управление
- Просмотр всех товаров в корзине
- Изменение количества товаров
- Удаление товаров
- Просмотр итоговой стоимости
- Сохранение на сервере и синхронизация с сессией пользователя

#### Оформление
1. Нажать "Оформить заказ"
2. Ввести адрес доставки
3. Проверить данные
4. Создать заказ

**Сохранение:** Данные корзины сохраняются в базе данных и доступны после входа в аккаунт

### 4️⃣ Избранное

#### Добавление
```javascript
POST /wishlist/{product_id}
```

#### Управление
- Просмотр избранных товаров
- Сетка избранного (аналогично каталогу)
- Быстрое добавление в корзину
- Удаление из избранного

#### Доступ
- Требует входа в аккаунт
- Сохраняется на сервере
- Синхронизируется с учётной записью пользователя

### 5️⃣ Заказы

#### Создание
```javascript
POST /orders/
{
  "shipping_address": "ул. Примерная, дом 1"
}
```

**Автоматический процесс:**
1. Товары берутся из корзины
2. Рассчитывается итоговая сумма
3. Создается заказ с адресом
4. Корзина очищается
5. Заказ появляется в истории

#### Просмотр
- История всех заказов
- Статус каждого заказа:
  - ⏳ **Ожидание** - заказ создан
  - ⚙️ **Обработка** - подготовка к отправке
  - ✅ **Доставлено** - получено
  - ❌ **Отменено** - не выполнен

#### Деталь заказа
- Номер заказа
- Список товаров с количеством
- Цена каждого товара
- Итоговая сумма
- Адрес доставки
- Дата создания

### 6️⃣ Интерфейс

#### Навигация
```
[Logo]  |  Главная  |  Товары  |  Избранное  |  Корзина  |  Заказы  |  👤 Вход
```

#### Модальные окна
- **Авторизация**: Регистрация/Вход/Профиль
- **Товар**: Полные детали и действия
- **Оформление**: Проверка заказа и адрес

#### Уведомления
```
✓ Успешно    (зеленый)
✗ Ошибка     (красный)
ℹ Инфо       (синий)
```

Автоматически исчезают через 3 секунды

#### Пустые состояния
- "Товаров не найдено" - при пустом результате поиска
- "Корзина пуста" - при пустой корзине
- "Избранное пусто" - при пустом избранном
- "Заказов нет" - при отсутствии заказов

## 🛠️ Техническая информация

### Архитектура

```javascript
// Основной объект приложения
const app = {
  // State
  currentUser: null,
  cart: [],
  wishlist: [],
  products: [],
  filters: {},
  
  // Methods
  async init() {},
  async loadProducts() {},
  async addToCart() {},
  // ... и многое другое
}
```

### Хранение данных
Корзина и избранное управляются через серверную базу данных и синхронизируются с учетной записью.

### API Integration

Все запросы используют **Fetch API** с credentials:
```javascript
fetch(url, {
  method: 'GET|POST|DELETE|PATCH',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'include', // для cookies
  body: JSON.stringify(data)
})
```

### CSS Grid & Flexbox

```css
/* Продукты */

  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 20px;
}

/* Элементы */
.product-actions {
  display: flex;
  gap: 8px;
}
```

## 📝 Примеры использования

### Добавить товар в корзину
```javascript
// Автоматически через UI
app.addToCart(productId)

// Результат: товар добавляется в корзину
// Уведомление: "Товар добавлен в корзину"
```

### Поиск товаров
```javascript
// Пользователь вводит в поле поиска
app.filters.search = "рубашка"
app.filterProducts()

// Результат: показываются только рубашки
```

### Создать заказ
```javascript
// Пользователь нажимает "Создать заказ"
// Вводит адрес доставки
app.createOrder()

// Результат: заказ создается, корзина очищается
// Уведомление: "Заказ создан успешно"
```

## 🐛 Решение проблем

### Не получается войти
- ✓ Убедитесь, что вы зарегистрировались
- ✓ Проверьте email и пароль
- ✓ Очистите cookies браузера

### Товары не загружаются
- ✓ Проверьте, что backend запущен
- ✓ Откройте http://localhost:8000/docs
- ✓ Проверьте консоль браузера (F12)

### Корзина не сохраняется
- ✓ Проверьте авторизацию пользователя
- ✓ Убедитесь, что backend запущен
- ✓ Проверьте доступ к API
